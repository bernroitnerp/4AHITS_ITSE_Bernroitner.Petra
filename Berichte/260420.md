# Reverse Shells Übungen

## Übung (netcat chat)
Verwende netcat um einen bidirektionalen Chat zwischen 2 unterschiedlichen Rechnern aufzubauen.


Was heißt *bidirektional*?

Ein bidirektionaler Chat bedeutet schlichtweg, dass Daten (in diesem Fall Textnachrichten) in beide Richtungen gleichzeitig fließen können. 
Beide Rechner fungieren sowohl als Sender als auch als Empfänger.


**Man hat 2 Rechner: Rechner A und Rechner B**

**Schritt 1: Rechner A öffnet den Port (Listening)**
Rechner A muss auf eine eingehende Verbindung warten. Wir wählen einen Port (z. B. 4444), der nicht von anderen Programmen belegt ist.

Gib folgenden Befehl ein:
```Bash
nc -l -p 4444
```

- ```-l```: Steht für Listen (Zuhören).
- ```-p 4444```: Gibt den Port an.

**Schritt 2: Rechner B verbindet sich**
Rechner B muss nun aktiv die Verbindung zu Rechner A aufbauen.

Gib folgenden Befehl ein:

```Bash
nc [IP-Adresse von Rechner A] 4444
```

**Wie man chattet**
Sobald Rechner B die Verbindung hergestellt hat, scheint im Terminal erst einmal nichts zu passieren. Das ist das Zeichen, dass die Leitung steht!

Tippe eine Nachricht auf Rechner A und drücke Enter. Sie erscheint sofort bei Rechner B.
Tippe eine Nachricht auf Rechner B und drücke Enter. Sie erscheint sofort bei Rechner A.

Um den Chat zu beenden, drücke auf einem der Rechner ```Strg + C```.


Umsetzung:

Ich habe den Port 1234 geöffnet und gelauscht.

```nc -l -p 1234```


Sarah hat sich mit dem Befehl ```nc 192.168.11.51 4444```

Dazugehängt und mir Hallo geschrieben.
Ich konnte darauf antworten.

## Übung (netcat data send)
Übertrage, per netcat, den Inhalt einer lokalen Text-Datei an einen anderen Schüler (auf einem anderen Rechner). Soll am Zielsystem in einer Textdatei gespeichert werden.


**1. Empfänger (Listener)**
Er muss zuerst den "Eimer" aufstellen, um die Daten aufzufangen:

```nc -l -p 4444 > empfangen.txt```

```>``` leitet alles, was über Netcat reinkommt, direkt in die Datei empfangen.txt.

<img width="405" height="60" alt="image" src="https://github.com/user-attachments/assets/19c39b86-88b1-43ba-93a8-024afd74d340" />


**2. Sender**
Du schickst die Datei jetzt durch die Leitung:

Bash
```nc [IP-PC_B] 4444 < deine_datei.txt```

```<``` liest den Inhalt von deine_datei.txt und "füttert" damit Netcat.

Davor muss der Host den Port geöffnet haben, sonst funktioniert es nicht!

## Übung (netcat banner grabbing II)
Erweitere das Kommando aus der vorhergehenden Übung so, dass die HTTP-Anfrage (Request) nicht über stdin eingegeben werden muss sondern per pipe | an nc übergeben wird.

Hinweis:

Die leere Zeile ist im HTTP Protokoll als die Zeichenfolge ```CRLF``` definiert.

CR (Carriage Return): Steuerzeichen mit ASCII-Wert dezimal 13 oder hexadezimal ```0x0D``` – ```\r``` in Programmen.
LF (Line Feed): Steuerzeichen mit ASCII-Wert dezimal 10 oder hexadezimal ```0x0A``` – ```\n``` in Programmen.
Verwende das Tool ```printf``` (statt ```echo```) um ```\r\n``` verwenden zu können.


```printf "HEAD / HTTP/1.0\r\n\r\n" | nc example.com 80```

- ```HEAD / HTTP/1.0``` → Request
- ```\r\n\r\n``` → zwei Leerzeilen (wichtig!)

**Auf Metasploitable:**
```printf "HEAD / HTTP/1.0\r\n\r\n" | nc 192.168.11.52 80```

## Übung (netcat port scanning)
Mit netcat kann man feststellen welche Ports auf einem Zielsystem an einen Dienst gebunden sind, diese Ports werden als offen bezeichnet.

*Verwende den Scanning-Modus (```-z```) mit einer Zeitbegrenzung (```-w 1```) und gib den Portbereich von 1 bis 1000 an um festzusellen welche Ports in Metasploitable geöffnet sind.*

*Allgemein:*
```nc -z -w 1 <IP> 1-1000```

**Metasploitable:**
```nc -z -w1 192.168.11.52 1-1000```

Bedeutung:
- ```-z``` → Scan-Modus (kein Daten senden)
- ```-w1``` → Timeout 1 Sekunde
- ```1-1000``` → Portbereich
