# portmapper/nfs vulnerability

## Übung (ssh auf Metasploitable)
Als Vorbereitung für die folgende Übung.

*Konfiguriere passwortloses ssh Login auf Metasploitable mit Hilfe eines ssh Public/Private keys.*

Betrachte besonders die notwendigen Änderungen in ~/.ssh/authorized_keys, diese Datei muss in der nachfolgenden Übung manipuliert werden.

- Hinweis: der ssh-server auf Metasploitable unterstützt nur Schlüssel im RSA Format (nicht die neueren default Formate wie ed255…).
  - Der Kali ssh Client lehnt die Verbindung mit diesem alten Schlüssel standardmäßig ab, aber man kann es mit Optionen wieder erzwingen.
  - Bei ssh Verbindungsproblemen hilft häufig die Verwendung der Verbose Optionen des ssh clients (```-v```, ```-vv``` oder ```-vvv```).

**Inhalt von authorized_keys**

So sieht der Inhalt konkret aus
Hier ist ein Beispiel, wie die Datei aussehen könnte, wenn zwei verschiedene Personen oder Geräte Zugriff haben:

Plaintext/ Beispiel:
```
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIOm6pX... user@laptop-zuhause
ssh-rsa AAAAB3Nza1yc2EAAAADAQABAAABgQCv9... admin@firma.de
```

**Inahlt RSA-Key**

Struktur eines RSA-Eintrags
Auch hier gilt: Alles muss in einer einzigen Zeile stehen.

- Typ: ssh-rsa
- Key: Ein riesiger Base64-kodierter String, der fast immer mit AAAAB3Nza1yc2E... beginnt.
- Kommentar: (Optional) z. B. nutzer@rechner

  
Plaintext/ Beispiel:
```
ssh-rsa AAAAB3Nza1yc2EAAAADAQABAAABAQCv9uH6...[viele weitere Zeichen]...5jL9pZ4f user@workstation
```

IP-Adresse von meinem Metasploitable: 192.168.168.132


**Generierung eines RSA-Schlüsselpaars auf Kali**

```ssh-keygen -t rsa -b 2048```

Erklärung:
- ```-t rsa``` → Schlüsseltyp RSA
- ```-b 2048``` → Schlüssellänge 2048 Bit

Standardmäßig werden die Schlüssel gespeichert unter:
- ```~/.ssh/id_rsa``` (Private Key)
- ```~/.ssh/id_rsa.pub``` (Public Key)

- <img width="640" height="359" alt="image" src="https://github.com/user-attachments/assets/dc61b146-082a-4051-ac2a-3217fdb70a15" />

**Schlüssel übertragen**

```ssh-copy-id -i ~/.ssh/id_rsa.pub -o HostKeyAlgorithms=+ssh-rsa -o PubkeyAcceptedAlgorithms=+ssh-rsa msfadmin@192.168.168.132```

**Kali Optionen - SSH-Verbindung mit alten RSA-Algorithmen erzwingen:**

Du kannst die entsprechenden Optionen direkt beim ssh-Befehl mitgeben. Du musst sowohl den Schlüssel-Algorithmus als auch die Authentifizierungsmethode explizit erlauben:

```ssh -o HostKeyAlgorithms=+ssh-rsa -o PubkeyAcceptedAlgorithms=+ssh-rsa nutzer@ziel-ip```

in meinem Fall:
```ssh -o HostKeyAlgorithms=+ssh-rsa -o PubkeyAcceptedAlgorithms=+ssh-rsa root@192.168.168.132```

*Mit -vvv erhält man detaillierte Debug-Ausgaben zur Authentifizierung.*
```ssh -vvv -o HostKeyAlgorithms=+ssh-rsa -o PubkeyAcceptedAlgorithms=+ssh-rsa msfadmin@192.168.168.132```
