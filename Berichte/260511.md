# Vorbereitung der Übungsumgebung
Betrachte folgendes Skript und finde heraus was es macht. Anschließend führe das Skript als User kali mit sudo aus, um die Laborumgebung einzurichten:


```
#!/bin/bash

# Gruppen anlegen
groupadd music
groupadd wrestle
groupadd emperors

# User anlegen und Gruppen zuweisen
useradd -m -G music elvis
useradd -m -G music,wrestle blondie
useradd -m -G wrestle hogan
useradd -m -G emperors nero

# Passwörter auf 'kali' setzen
echo "elvis:kali" | chpasswd
echo "blondie:kali" | chpasswd
echo "hogan:kali" | chpasswd
echo "nero:kali" | chpasswd

# Lab-Struktur vorbereiten
mkdir -p /tmp/lab/gigs /tmp/lab/drafts
echo "Draft" > /tmp/lab/drafts/lyrics.txt
echo "Plans" > /tmp/lab/wrestle_plans.txt
chown -R root:root /tmp/lab
chmod -R 755 /tmp/lab
```

# Übung (Analyse und Symbolic Mode)
1. Rechte auslesen: Ermittle die Permissions, den User Owner und den Group Owner der Datei /tmp/lab/wrestle_plans.txt?

```ls -l /tmp/lab/wrestle_plans.txt```
<img width="555" height="64" alt="image" src="https://github.com/user-attachments/assets/3f3f02b5-4d69-4c7f-8ee4-b49381b73ef2" />

-rwxr-xr-x

- rwx r-x r-x (immer 3er Gruppen)

- ```-``` = nicht directory
- ```rwx``` = Benutzer darf lesen, schreiben und ausführen
- ```r-x``` = Gruppe darf nur lesen und ausführen
- ```r-x``` = others dürfen auch nur lesen und ausführen


2. Private Files: Erstelle eine Datei notes.txt. Ändere die Permissions so, dass der other-Klasse (o) das Read-Recht (r) entzogen wird. Nutze den symbolischen Modus von chmod.

*Datei erstellen:*

```touch notes.txt```

*Read-Recht für others entfernen:*

```chmod o-r notes.txt```

<img width="241" height="101" alt="image" src="https://github.com/user-attachments/assets/a7bacff4-6dcb-4ee6-a9bc-470b234c298f" />

**Nebeninfo**
- ein + fügt immer die Rechte hinzu
- ein - entnimmt die Rechte

  

3. Group Sharing: Setze die Permissions für notes.txt so, dass die group (g) die Datei lesen (r) und schreiben (w) darf, während der user (u) alle Rechte behält und others (o) keinerlei Zugriff haben. Verwende für die Änderung einen einzigen chmod Aufruf.

*Ein einziger chmod-Befehl:*

```chmod u=rwx,g=rw,o= notes.txt```

# Übung (Directory Permissions und Octal Notation)
Oktale Umwandlung: Gib die chmod-Befehle mit Oktalwerten für folgende Zielzustände an:

Info:

```
Rechte	  Oktal
rwx	      7
rw-	      6
r--	      4
r-x	      5
---	      0
```


- rwx r-x r-x = 755
- rw- rw- --- = 660
- rwx --- --- = 700


*Pub-Verzeichnis:* Erstelle das Verzeichnis /home/kali/pub. Setze mit einem Befehl in oktaler Notation die Rechte so: Der Besitzer darf alles (7), die Gruppe darf den Inhalt auflisten und in das Verzeichnis wechseln (5), alle anderen dürfen nur in das Verzeichnis wechseln, aber den Inhalt nicht auflisten (1).

=>

chmod 751
rwx rw- --x

*Verzeichnis erstellen:*

```
mkdir /home/kali/pub

```

*Recursive Lockdown:* Du möchtest das Verzeichnis /tmp/lab/drafts/ und alle darin enthaltenen Dateien rekursiv für die Gruppe music lesbar (r) und beschreibbar (w) machen, während others keinen Zugriff haben. 
Welcher Befehl ist dafür nötig?

Wenn die Dateien noch nicht zur Gruppe music gehören, braucht man ```chgrp``` (Info von ChatGPT)

```
chgrp -R music /tmp/lab/drafts/
chmod -R g=rwX,o= /tmp/lab/drafts/
```
